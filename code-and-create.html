Reorienting ourselves with computers


    

    
Do you have a computer/desktop/laptop at home? Do you use it every day?
    
Today, we will get reoriented with our computers and write poetry with their logic. 
    
We will write something called folder poetry, a practice that my mentor, Melanie Hoff, made up, but is also completely real. By repurposing the common practice of computer folder organization as a poetic form. By naming and nesting folders and files, we can create narratives, rhythmic prose, and choose-your-own-adventure poetry.
    
We will do a short exercise to write folder poetry, and navigating the command line interface using a language called Bash.
    
Mac terminal's default language is Zsh, which is an updated version of Bash. 
    
  • To switch back to the basic Bash, enter this in your command line: 

    
chsh -s /bin/bash
  
    








    
    
what you might see when you open up terminal
    




 





    






    
As you look at the terminal you should see...mostly nothing. This is as it should be. 
     





    
 There are no notifications, no pop-up with daily tips, no indications of how you should start.
   





      So, to start: do nothing. Notice how the terminal, too, does nothing. 
   





   
No prompt will appear asking if you need a little more time, no chat window with a cheery reminder of commands to try. The terminal just waits. It has no expectations, no motives. It has no emotion at all. 
    




  




 







   The bit that keeps the terminal from being a full on blank page is a line of cryptic text and $. The exact line will be different for each terminal. You may see your name and a ~, and perhaps the name of your computer. 
    




 




 





    
What are you seeing now?

    




 




 





    
This is what‚Äôs known as the prompt, and it‚Äôs where you can enter in text for your computer to respond. It is the ‚Äúline‚Äù part of command line.
    


    
To use the terminal is to engage in a dialogue with your computer. You will ask it to do something and it will do it. Most of the time it gives no outward response, just moves quickly and diligently through the task and waits silently for your next ask. If you ask a question of it, it will print its answer. If it doesn‚Äôt understand you or encountered an issue, it will print out its problem as best as it can articulate.

    



    
Washen you see the prompt, you can know the dialogue is happening, that the computer is waiting for the next thing you type.
     




 




 





      
Let‚Äôs go ahead and start this dialogue.

     




 




 





    
Type the following into your terminal and press enter:

    
$ hello
 
    





    
you should see this response:

    
bash: hello: command not found

    





    
This is a confusing response, but notice that the computer did not say that you said anything wrong. Rather, you typed in what it thought is a command, but it couldn't match it with an action.

    





    
In other words: the computer doesn‚Äôt understand what you‚Äôre saying.

Other languages in an .html file

    
Some elements, mainly style and script, aren‚Äôt visible but contain code in other languages that change how the page looks and works!
    
This includes CSS (stands for Cascading Style Sheets) and Javascript (or js)
    
When we create a website, we need to specify what type of information they are conveying -- and they each speak in a slightly different language too!
    
There are two ways to separate them:
      
  • Within your HTML file, separate your CSS with <style>...</style> and Javascript with <script>...</script>
      
    Note that <style>...</style> goes in <head>...</head> and <script>...</script> goes in <body>...</body> at the very end 
      
    You can also insert CSS using style="", but this is only recommended for simple changes, or it might make your file really messy. 
        
    Here is an example:
          
     <p style="color:green"> hello! </p>

      What is CSS?
      

      CSS stands for Cascading Style Sheets. It describes how HTML elements are to be displayed on screen, paper, or in other media

      CSS saves a lot of work. It can control the layout of multiple web pages all at once
  

      
  
      It also makes your website have personality and reflect who you are!




      

      CSS can be added in three different ways:

      Inline - by using the style attribute inside HTML elements:
    
      <p style="color:pink">Hello!<p>
      
      Hello!

      Internal - by using a <style> element in the <head> section

      External - by using a <link> element to link to an external CSS file

      This is the template for a CSS style element:
  
      
h1 {
      
  color: blue;
      
  font-family: verdana;
      
  font-size: 300%;
      
}
      
p {
      
  color: red;
      
  font-family: courier;
      
  font-size: 160%;
      
}      

      


      



      

      You can create a custom class for different sections for the website.

      Class is represented in CSS by .CLASSNAME (period + class name)

      

.center {
      
  text-align: center;
      
  color: red;
      
}
      

      

      Then, you can add it into your html

      
  <p class="center">Red and center-aligned paragraph.</p>
      

      Red and center-aligned paragraph.
       



      


      Class is reusable, this means you can use the same class selector multiple times within the same page
      

      What if you want something more specific
      

      id is a different type of selector that can only be used once within the same page
      

      id is indicated by #IDNAME(# + id name -- note that id name cannot start with a number!)

      everything else works the same way as class:
  
      

#firstParagraph {
      
  text-align: right;
      
  color: yellow;
      
}
      

      

      Adding it to html...

      
  <p id="firstParagraph">Yellow and right-aligned paragraph.</p>
      

      Yellow and right-aligned paragraph.
